React Basics for Beginners

What is React? ▼

React is a JavaScript library for building user interfaces, created by Facebook.

  • Makes it easy to create interactive web applications.
  • Uses a component-based architecture (reusable pieces of UI).
  • Efficiently updates the DOM when data changes.
Why React?
  • Component reusability
  • Virtual DOM for performance
  • Large ecosystem and community
  • Declarative programming

Components ▼

Components are the building blocks of React applications.

  • Think of them as custom HTML elements.
  • Can be reused throughout your app.
  • Can contain other components.
Example Component: 
function Welcome() {
  return (
    <div>
      <h1>Hello, React!</h1>
      <p>Welcome to my first component.</p>
    </div>
  );
}
Using the Component: 
<Welcome />

JSX ▼

JSX is a syntax extension for JavaScript that looks like HTML.

  • Allows you to write HTML-like code in JavaScript.
  • Gets compiled to regular JavaScript.
  • Makes components easier to read and write.
JSX vs Regular JavaScript: 
// JSX
const element = <h1>Hello, world!</h1>;

// Regular JavaScript (what JSX compiles to)
const element = React.createElement('h1', null, 'Hello, world!');
JSX Rules:
  • Must have a single parent element (or use React.Fragment).
  • Use className instead of class.
  • Use htmlFor instead of for.
  • Use camelCase for attributes.

Props ▼

Props (properties) are how you pass data from parent to child components.

  • Props are read-only (cannot be changed by the child).
  • Can be any type of data (strings, numbers, objects, functions).
Example: 
// Parent component
function App() {
  return (
    <div>
      <Greeting name="Alice" age={25} />
      <Greeting name="Bob" age={30} />
    </div>
  );
}

// Child component
function Greeting(props) {
  return (
    <h2>Hello, {props.name}! You are {props.age} years old.</h2>
  );
}
Destructuring Props: 
function Greeting({ name, age }) {
  return (
    <h2>Hello, {name}! You are {age} years old.</h2>
  );
}

State ▼

State is data that can change over time in a component.

  • When state changes, the component re-renders.
  • State is private to the component.
  • Use useState hook to manage state.
Example with useState: 
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  );
}
useState Breakdown:
  • useState(0) - Creates state with initial value 0.
  • count - Current state value.
  • setCount - Function to update state.

Hooks ▼

Hooks are functions that let you "hook into" React features.

  • Must be called at the top level of your component.
  • Cannot be called inside loops, conditions, or nested functions.
Common Hooks:
  • useState - Manage state
  • useEffect - Handle side effects
  • useRef - Access DOM elements
  • useContext - Access context
useEffect Example: 
import { useState, useEffect } from 'react';

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);

  useEffect(() => {
    // Fetch user data when component mounts or userId changes
    fetch(`/api/users/${userId}`)
      .then(response => response.json())
      .then(data => setUser(data));
  }, [userId]);

  if (!user) return <div>Loading...</div>;

  return (
    <div>
      <h2>{user.name}</h2>
      <p>{user.email}</p>
    </div>
  );
}

Event Handling ▼

Event Handling in React is similar to HTML, but with camelCase names.

  • Use camelCase for event names (onClick, onChange).
  • Pass functions as event handlers.
  • Event handlers receive an event object.
Example: 
function Form() {
  const [name, setName] = useState('');

  const handleSubmit = (event) => {
    event.preventDefault();
    alert('Hello, ' + name + '!');
  };

  const handleChange = (event) => {
    setName(event.target.value);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        value={name}
        onChange={handleChange}
        placeholder="Enter your name"
      />
      <button type="submit">Submit</button>
    </form>
  );
}

Conditional Rendering ▼

Conditional Rendering allows you to show different content based on conditions.

Using if/else: 
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  } else {
    return <h1>Please log in.</h1>;
  }
}
Using ternary operator: 
function Greeting({ isLoggedIn }) {
  return (
    <h1>{isLoggedIn ? 'Welcome back!' : 'Please log in.'}</h1>
  );
}
Using logical AND: 
function Notification({ message }) {
  return (
    <div>
      {message && <p>{message}</p>}
    </div>
  );
}

Lists & Keys ▼

Lists in React are rendered using JavaScript's map() function.

Example: 
function TodoList() {
  const todos = ['Learn React', 'Build an app', 'Deploy to production'];

  return (
    <ul>
      {todos.map((todo, index) => (
        <li key={index}>{todo}</li>
      ))}
    </ul>
  );
}
Important Notes:
  • Always use a key prop when rendering lists.
  • Keys help React identify which items have changed.
  • Use unique, stable IDs when possible (not array indices).
Better Example with IDs: 
function TodoList() {
  const todos = [
    { id: 1, text: 'Learn React' },
    { id: 2, text: 'Build an app' },
    { id: 3, text: 'Deploy to production' }
  ];

  return (
    <ul>
      {todos.map(todo => (
        <li key={todo.id}>{todo.text}</li>
      ))}
    </ul>
  );
}
